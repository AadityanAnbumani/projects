## Getting Started 

### 1. The Brain: Arduino 🧠

For this project, I used an **Arduino UNO R4 Minima**.

<ImageZoom src="https://raw.githubusercontent.com/codedex-io/projects/refs/heads/main/projects/build-a-pocket-rave-with-arduino/assets/uno.jpeg" />

If you are completely new to electronics, you might be wondering: *What on earth is an Arduino?*

Think of an **Arduino** as a tiny, pocket-sized computer. Just like your laptop, a microcontroller has a processor to follow instructions, memory to store information, and connections that let it communicate with other electronic devices. The difference is that it's much smaller – about the size of a cracker!

However, it doesn’t have a built-in screen or a keyboard. It is completely blind and deaf until we plug things into it, and it doesn't know how to do anything until we give it a list of rules (our code). Once we type up our instructions and send them to the Arduino, it becomes the boss and tells all the connected lights, screens, and motors exactly what to do.

**Where to buy it:** [Official Arduino Store](https://store-usa.arduino.cc/products/uno-r4-minima), [Amazon](https://a.co/d/0b24YgYf)

**Pro Tip: Swapping the Brain!** You don't have to use the UNO R4! Microcontrollers are like different breeds of dogs – they might look a little different, but they all love to play fetch. You can also use an Arduino Nano ESP32 (which even has built-in Wi-Fi and Bluetooth!) or a Raspberry Pi Pico. As long as your new board has an I2C connection (more on that below!), you can throw the pocket rave.

### 2. The Tiny TV: OLED Display 📺

This Tiny TV is a super small screen where our rave will actually happen. Even though it's tiny, it's bright enough to display crisp animations, text, and graphics.

**OLED** stands for **O**rganic **L**ight-**E**mitting **D**iode. That's just a fancy name that means "a screen made of thousands of teeny-tiny lightbulbs that can turn on/off individually." This display technology is found in modern TVs, monitors, laptops, smartphones, etc.

Our display has a technical name: the **SSD1306 128x64 OLED (I2C)**. 😵‍💫

<ImageZoom src="https://raw.githubusercontent.com/codedex-io/projects/refs/heads/main/projects/build-a-pocket-rave-with-arduino/assets/oledwsong.jpeg" />

Let's break down that long name:

- SSD1306 is the display controller model number.
- 128x64 means the screen is made up of a grid of tiny lights. 128 across and 64 down. Total of 8,192.
- OLED is the display technology used by the screen (explained above).
- I²C (Inter-Integrated Circuit) is the communication method the Arduino uses to talk to the display.

Think of I²C as a messaging system. The Arduino drops instructions into the tube ("Turn on lightbulb at grid coordinate (10, 25)"), and the screen follows the order almost instantly. That's why we only need those two special wires: 

- SDA (the data tube) 
- SCL (the clock that keeps the deliveries on schedule)

The good news? You don't need to understand how this works! We'll use a library that handles all of it.

**Where to buy it:** [Amazon](https://a.co/d/00ZNDdmh).


### 3. The Power Line: USB-C Cable ⚡

To upload code from your laptop to the Arduino (and power it), we'll need a USB-C cable, which is usually included with the Arduino.

A regular USB-C phone charging / data cable works perfectly! Just make sure it's a data cable – some USB-C cables are power-only and can't transfer data.

**Where to buy it:** [Adafruit](https://www.adafruit.com/product/4199)


### 4. The Messengers: Jumper Wires 🪢

Finally, we need 4 Female-to-Male jumper wires. These are essentially strings of metal encased in colored plastic. They carry electricity and secret messages back and forth between the Brain and the Tiny TV.

**Where to buy it:** [Adafruit](https://www.adafruit.com/product/1953), [Amazon](https://a.co/d/042vadtF)

### Default Code

The default code inside the code editor is the "Bare Minimum" sketch:

```cpp
void setup() {
  // put your setup code here, to run once:
  
}

void loop() {
  // put your main code here, to run repeatedly:
  
}
```

- `setup()` runs once when the board powers on or resets. Use it to initialize the hardware.
- `loop()` runs repeatedly as long as the board has power. This is where the main program logic goes.

We'll come back to this in a bit!

## Connecting the Dots (The Wiring)

How do we get the Brain to talk to the TV? We connect them with our 4 messenger wires! 

Here is a super easy way to remember the wiring:

<video width="100%" autoPlay loop muted playsInline>
  <source src="https://raw.githubusercontent.com/codedex-io/projects/refs/heads/main/projects/build-a-pocket-rave-with-arduino/assets/wkConnection (1).mp4" type="video/mp4" />
</video>

- **VCC** → **3.3V** (Power): VCC powers the display, just like a TV needs electricity to turn on. It connects to the Arduino’s 3.3-volt power pin.
- **GND** → **GND** (Ground):  GND gives electricity a safe path back, completing the circuit so everything works safely. Think of it like a pair of shoes keeping everything grounded.
- **SDA** → **A4** (Data): SDA stands for Serial Data and it carries text and pictures to the display.
- **SCL** → **A5** (Clock): SCL stands for Serial Clock. It keeps the rhythm and timing for the data.

Here's an easy mememory trick: Just count the letters!

- SDA is "data" = 4 letters = Pin A4
- SCL is "clock" = 5 letters = Pin A5

Once you remember that, you'll never have to look up the wiring again.

## Writing the Magic Spells (The Code)

Now for the fun part: teaching the Brain how to throw a party using C++!

### 1. Giving the Brain a Dictionary

At the very top of the program, even above `setup()`, add:

```cpp
#include <Wire.h>
#include <Adafruit_GFX.h>
#include <Adafruit_SSD1306.h>

#define SCREEN_WIDTH 128
#define SCREEN_HEIGHT 64
```

Think of these like giving the Arduino a dictionary. Instead of teaching the Arduino what a "screen" is from scratch, we just hand it a book that says, "Here is how you talk to the tiny TV."

The two `#define` lines give constant values simple nicknames. Instead of typing the number 128 a bunch of times, we just tell the computer: "Whenever I say `SCREEN_WIDTH`, I mean 128."

These definitions specify the dimensions of a display (often an OLED screen):

- `SCREEN_WIDTH`: the display is 128 pixels wide
- `SCREEN_HEIGHT`: the display is 64 pixels tall

```output
          128 pixels
+----------------------------+
|                            |
|                            |  64 pixels
|                            |
+----------------------------+
```

Next, add:

```cpp
#define OLED_RESET    -1 

Adafruit_SSD1306 display(SCREEN_WIDTH, SCREEN_HEIGHT, &Wire, OLED_RESET);
```

This code sets up your main display object with the screen's dimensions and I2C communication, while the `-1` tells the Arduino that the OLED doesn't use a dedicated reset wire.

### 2. Drawing with 1s and 0s

If you squint your eyes and look at the 1s in the array below, you can maybe see the shape of a heart! ♡

We save this "picture" under the name `heart_bmp[]` so we can stamp it on the screen later:

```cpp
const unsigned char heart_bmp[] PROGMEM = {
  0b01010000,
  0b11111000, 
  0b11111000,
  0b01110000,
  0b00100000
};
```

Remember, computers only understand two numbers: 1 (on) and 0 (off). 

To draw the little floating hearts on our screen, we literally draw a picture using 1s and 0s: 

- `1` tells the screen to turn a tiny lightbulb on.
- `0` tells the screen to leave it dark.

By this point, your program should now look like:

```cpp
#include <Wire.h>
#include <Adafruit_GFX.h>
#include <Adafruit_SSD1306.h>

#define SCREEN_WIDTH 128
#define SCREEN_HEIGHT 64
#define OLED_RESET    -1 

Adafruit_SSD1306 display(SCREEN_WIDTH, SCREEN_HEIGHT, &Wire, OLED_RESET);

const unsigned char heart_bmp[] PROGMEM = {
  0b01010000,
  0b11111000, 
  0b11111000,
  0b01110000,
  0b00100000
};

void setup() {
  // put your setup code here, to run once:
  
}

void loop() {
  // put your main code here, to run repeatedly:
  
}
```

### 3. The Initial Setup 

The `setup()` block is the initial boot sequence of our screen. 

Inside it, let's add:

```cpp
void setup() {
  if(!display.begin(SSD1306_SWITCHCAPVCC, 0x3C)) {
    for(;;); 
  }
  
  // LARGE FONT
  display.setTextSize(2);
  display.setTextColor(SSD1306_WHITE);
}
```

`if` statement: The Arduino is knocking on the screen's door to wake it up. The `0x3C` is just the screen's "street address" so the Arduino knows exactly where to send your graphics.

Now that the screen is awake, `.setTextSize()` and `.setTextColor()` set the font size and color.

### 4. The Ultimate Secret: millis() vs. delay()

The `loop()` block is the main program of our Arduino. Everything inside runs over and over again!

Inside it, let's add:

```cpp
display.clearDisplay();
```

It acts as an eraser that wipes the screen completely clean before drawing the next frame of your animation so nothing gets messy or overlaps.

Then add more:

```cpp
void loop() {
  display.clearDisplay();
  
  unsigned long timeNow = millis();

  // 130 BPM pacing: 1.8 second chunks for a 3.6 second loop
  unsigned long songLoop = timeNow % 3600; 
  int phase = songLoop / 1800; 
}
```

When beginners first learn Arduino, they often use a function called `delay()` whenever they want the microcontroller to wait before doing the next task. But `delay()` is like telling the Arduino to take a nap! While the Arduino is asleep, it can't do anything else. Not update the lyrics, not make the stick figure dance, nothing. Everything freezes until the delay is over.

To solve this, we use `millis()`. Instead of putting the Arduino to sleep, `millis()` tells the Arduino to constantly check a stopwatch. It says, "Keep looking at your watch! When 1.8 seconds pass, change the lyrics. When 230 milliseconds pass, make the guy dance!" This is called non-blocking code, and it lets our stick figure dance smoothly while the song lyrics scroll by!

The `timeNow` variable acts like a continuous stopwatch, capturing the exact number of milli sec since we are using the `millis()` function. 

While the `songLoop` uses the `%` modulo operator to chop that never ending stopwatch into perfectly repeating 3.6 second (3600 ms) – means the stopwatch resets back to 0 after 3.6s and goes back to the first line of the given lyrics.

Finally, `phase` divides the current position within the 3.6-second loop, each lasting 1.8 seconds. Each section corresponds to one line of the song lyrics, so every 1.8 seconds the displayed lyric advances to the next line.

I came up with 3.6-second loop duration was determined experimentally. I adjusted the timing through several rounds of trial and error until the lyric transitions matched the rhythm and tempo of the song.


### 5. The Floating Particle Effects (The Hearts)

To make the background look like a real party, we stamp our `heart_bmp` onto the screen. But we want two of em, and we want them to float upwards!
  
On our screen, a Y-coordinate of 64 is the very bottom, and 0 is the very top.

```cpp
void loop() {
  ...
  
  int heart1_Y = 64 - ((timeNow / 40) % 70);
  int heart2_Y = 64 - (((timeNow + 1500) / 40) % 70); 
  
  display.drawBitmap(120, heart1_Y, heart_bmp, 5, 5, SSD1306_WHITE);
  display.drawBitmap(95,  heart2_Y, heart_bmp, 5, 5, SSD1306_WHITE);
  
  ...
}
```

The OLED screens map their Y-axis upside down so 64 is the floor and we subtract it with the slowed-down timer `/ 40` to make the hearts float upwards. The `% 70` acts as the teleporter and it instantly snaps the heart back to the floor once it reaches the top so the party never ends.

To make it look natural, we sneak a `+ 1500` into the second heart's timer to stagger its launch so they aren't floating right on top of each other. Finally, the `120` and `95` lock them into their own separate vertical dance lanes on the far right side of the screen so they don't crash into your stick figure!

- Click the **Upload** button (the right-pointing arrow) to compile and upload the sketch to your Arduino.

### 6. Choreographing the Dance

When people see the little guy dancing on the screen, they usually think it's a tiny video or a GIF. Nope! We are literally drawing the figure from scratch, 60 times a second, using a digital Etch-A-Sketch. 🕺

Snippets on how you can construct your own cute stick figure or even [tung tung tung sahur](https://en.wikipedia.org/wiki/Tung_Tung_Tung_Sahur). ;-)

```cpp
display.drawCircle();
display.drawLine();
```

<ImageZoom src="https://raw.githubusercontent.com/codedex-io/projects/refs/heads/main/projects/build-a-pocket-rave-with-arduino/assets/jumping-man.gif" style={{ width: "30%", height: "auto" }} />

Instead of an image, we use math to draw shapes. How does he move? Remember our `millis()` stopwatch? Every 230 milliseconds, a variable called `danceBeat` flips between 0 and 1.

```cpp
int danceBeat = (timeNow / 230) % 2;
int guyY = 25 + (danceBeat == 1 ? -4 : 0);
```                                                                                                       

I love choreographing dances, so I had to make my stick figure bounce to the exact rhythm of the song. The song is 130 Beats Per Minute (BPM). Doing a little bit of math, that means every 230 milliseconds, a fast, club-style eighth-note beat hits!

Every time the stopwatch hits that 230ms mark, we tell the stick figure to throw his arms up and bounce his Y-coordinate up by 4 pixels.

No matter if he is at the peak of his jump or grounded in a squat, his head and spine stay the exact same. We anchor them directly to our bouncing `guyX` and `guyY` variables.

```cpp
// The Head
display.drawCircle(guyX, guyY, 4, SSD1306_WHITE); 
// The Spine
display.drawLine(guyX, guyY + 4, guyX, guyY + 16, SSD1306_WHITE);
```

Then, we use a simple `if/else` statement acting as our rapid-fire frame switcher for his limbs:

**Beat 1 (The Jump)**: We draw four lines spreading straight outward for his arms and legs.

**Beat 0 (The Squat)**: We bring his arms down and bend his knees! Bending a joint means drawing two distinct lines for each leg: one line from the hip to the knee, and a second line connecting the knee to the foot.

Here is the complete code that handles the timing, the bounce, and every single coordinate needed to make him groove:

```cpp
void loop() {
  ...

  // He ONLY drops in during specific lyric phases
  bool showStickFigure = (phase == 1);

  if (showStickFigure) {
    // 130 BPM Math: 230ms for a fast, club-style eighth-note bounce!
    int danceBeat = (timeNow / 230) % 2; 
    int guyX = 110; 
    int guyY = 25 + (danceBeat == 1 ? -4 : 0);  
    
    // Draw the Head and Spine (These stay the same every frame)
    display.drawCircle(guyX, guyY, 4, SSD1306_WHITE); 
    display.drawLine(guyX, guyY + 4, guyX, guyY + 16, SSD1306_WHITE); 

    if (danceBeat == 1) {
      // POSE 1: THE JUMP (Arms up, legs wide)
      display.drawLine(guyX, guyY + 8, guyX - 10, guyY - 2, SSD1306_WHITE);       // Left Arm
      display.drawLine(guyX, guyY + 8, guyX + 10, guyY - 2, SSD1306_WHITE);       // Right Arm
      display.drawLine(guyX, guyY + 16, guyX - 8, guyY + 28, SSD1306_WHITE);      // Left Leg
      display.drawLine(guyX, guyY + 16, guyX + 8, guyY + 28, SSD1306_WHITE);      // Right Leg
    } else {
      // POSE 2: THE SQUAT (Arms down, knees bent)
      display.drawLine(guyX, guyY + 8, guyX - 8, guyY + 16, SSD1306_WHITE);       // Left Arm
      display.drawLine(guyX, guyY + 8, guyX + 8, guyY + 16, SSD1306_WHITE);       // Right Arm
      
      display.drawLine(guyX, guyY + 16, guyX - 6, guyY + 24, SSD1306_WHITE);      // Left Leg (Thigh to knee)
      display.drawLine(guyX - 6, guyY + 24, guyX - 2, guyY + 30, SSD1306_WHITE);  // Left Leg (Knee to foot)
      
      display.drawLine(guyX, guyY + 16, guyX + 6, guyY + 24, SSD1306_WHITE);      // Right Leg (Thigh to knee)
      display.drawLine(guyX + 6, guyY + 24, guyX + 2, guyY + 30, SSD1306_WHITE);  // Right Leg (Knee to foot)
    }
  }
```


### 7. The Digital Teleprompter

Now for the final piece... _drumroll, please_ 🥁... the lyrics! This is where you get to customize the code so it matches your favorite song. Mine is ["On The Floor"](https://www.youtube.com/watch?v=t4H_Zoh7G5A) by J.Lo (feat. Pitbull):

> _If you go hard, you gotta get on the floor_  
> _If you're a party freak, then step on the floor_  
> _If you're an animal, then tear up the floor_  

We use an `if` / `else if` statement like a movie script. The Arduino checks our `millis()` stopwatch to see what "phase" of the song we are in, and prints the correct lyrics to the screen!

**Want to swap in your own track?** Just change the printed text inside each block, add more `else if` phases to make the song longer, and tweak that `1800` (1.8 seconds) phase timer we made earlier to perfectly match your new song's BPM.

```cpp
void loop() {
  ...
  
  if (phase == 0) {
    display.setCursor(0, 8);
    display.println("If you"); 
    display.println("go hard"); 
    display.println("you gotta");
  } else if (phase == 1) {
    display.setCursor(0, 8);
    display.println("get on"); 
    display.println("the"); 
    display.println("floor");
  }
  // TO ADD MORE LYRICS: Just copy the block above, change 'phase == 1' to 'phase == 2', and so on to add more lyrics... 
  // and type your new lyrics inside the display.println() commands!
  
  display.display();
}
```
